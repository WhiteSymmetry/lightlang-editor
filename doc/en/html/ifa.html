About system "Integradable Friend Applications"

The system is development of two persons: Devaev Maxim and Tikhonov Sergey.
The base of this system is usage xml, as information about friend program. There are two versions of IFA: first is written on PyQt4 by Devaev Maxim and another is written on Qt4 + C++ by Tikhonov Serget. Version on PyQt4 is wider, than on Qt4 + C++, becouse of more high needs. The LightLang Editor uses second version, that's why here, more in detail is describing second version:
IFA handles such xml view:
<app>
    <title>Common name</title>
    <title lang="<language>">Name on the language</title>
    <description>Common description</description>
    <description lang="<language>">Description on the language</description>
    <icon>Path to icon</icon>
    <path>Path to program</path>
</app>
 IFA memorize all these arguments and when is called special function, in which as argument is given QAction, this QAction is filled by arguments.
Such xml-files can be a lot of, but main is, that they situate in special directory, which the system handles.
